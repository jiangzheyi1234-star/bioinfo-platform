## 2. API 通讯流程详解

### 2.1 前端 HTTP 请求流程

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                    前端 → 后端 HTTP 流程                                 │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                         │
│  前端 (Next.js/React)                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │                                                                                 │   │
│  │  Step 1: API Base URL                                                           │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  detection_workspace_utils.ts                                            │   │   │
│  │  │                                                                          │   │   │
│  │  │  export function apiBase(): string {                                     │   │   │
│  │  │    const raw = process.env.NEXT_PUBLIC_API_BASE                          │   │   │
│  │  │                  || "http://127.0.0.1:8765";  ◄── 默认地址              │   │   │
│  │  │    return raw.trim().replace(/\/+$/, "");                                │   │   │
│  │  │  }                                                                       │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  │  Step 2: Fetch Request                                                          │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  const resp = await fetch(`${apiBase()}/api/v1/projects`);              │   │   │
│  │  │  const data = await readJsonOrThrow(resp);                               │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  │  Step 3: Data Parsing                                                           │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  const items = data.items.map(toProject)                                 │   │   │
│  │  │                .filter(item => !!item);                                  │   │   │
│  │  │  setProjects(items);                                                     │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                          │                                              │
│                                          │ HTTP GET http://127.0.0.1:8765/api/v1/projects
│                                          ▼                                              │
│  后端 (FastAPI)                                                                         │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │                                                                                 │   │
│  │  Step 4: Route Handler                                                          │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  @app.get("/api/v1/projects")                                            │   │   │
│  │  │  async def list_projects(...) -> dict[str, Any]:                         │   │   │
│  │  │      return {"items": _runtime().list_projects(...)}                     │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  │  Step 5: Runtime Service                                                        │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  RuntimeService.list_projects()                                          │   │   │
│  │  │  └─ self._project_manager.list_projects()                                │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  │  Step 6: Database Query                                                         │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  SQLite Query: SELECT * FROM projects ORDER BY created_at DESC          │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  │  Step 7: JSON Response                                                          │   │
│  │  ┌─────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │  { "items": [{                                                          │   │   │
│  │  │      "project_id": "...",                                                │   │   │
│  │  │      "name": "Project Name",                                             │   │   │
│  │  │      "status": "active",                                                 │   │   │
│  │  │      "created_at": 1234567890                                            │   │   │
│  │  │  }] }                                                                    │   │   │
│  │  └─────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                 │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

## 6. 关键通讯模式总结

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                    通讯模式总结                                          │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                         │
│  模式 1: RESTful API (HTTP)                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │  用途: 查询操作、状态获取、配置管理                                             │   │
│  │  示例: GET /api/v1/projects, POST /api/v1/executions                            │   │
│  │  频率: 按需调用                                                                 │   │
│  │  特点: 同步请求/响应，JSON 格式                                                 │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
│  模式 2: Event Stream (轮询)                                                            │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │  用途: 实时执行状态更新                                                         │   │
│  │  端点: GET /api/v1/events/executions?after_seq={seq}                            │   │
│  │  频率: 每 1-2 秒轮询                                                            │   │
│  │  特点: 长轮询，按序列号增量获取                                                 │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
│  模式 3: Signal/Event (后端内部)                                                        │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │  用途: 核心服务间异步通信                                                       │   │
│  │  实现: pyqtSignal (core/qt_compat.py)                                           │   │
│  │  信号: execution_started, execution_completed, execution_failed                 │   │
│  │  特点: 线程安全，观察者模式                                                     │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
│  模式 4: SSH 远程调用                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │  用途: 在远程服务器执行生物信息学工具                                           │   │
│  │  协议: SSH2 (Paramiko)                                                          │   │
│  │  约束: 单队列串行执行 (SSHService.run())                                        │   │
│  │  特点: 命令通过 screen 在后台分离执行                                           │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
│  模式 5: SQLite 本地存储                                                                │
│  ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│  │  用途: 项目数据、执行记录、配置持久化                                           │   │
│  │  位置: ~/.h2ometa/projects/{project_id}/project.db                              │   │
│  │  访问: 通过 ProjectManager, DatabaseService                                     │   │
│  │  特点: 单文件数据库，零配置                                                     │   │
│  └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```
